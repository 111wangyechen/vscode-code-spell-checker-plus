# 用户故事

## 1. 技术专业人员拼写检查需求

**用户角色**：软件开发者、数据科学家、技术文档编写者
**场景**：在编写代码、文档或技术报告时，经常需要使用特定的技术术语（如"tensorflow"、"kubernetes"）、日语词汇或包含拼音的中文名称
**痛点**：
- 通用拼写检查工具无法识别专业技术术语，导致误报
- 无法正确处理日语字符和拼音拼写
- 多个拼写错误同时存在时，拼写建议功能失效
**期望**：
- 拼写检查器能够识别常见技术术语
- 支持日语和拼音拼写检查
- 在存在多个拼写错误时仍能提供准确的修复建议
- 提供实时拼写错误检测和修复建议

## 2. 多语言内容创作者需求

**用户角色**：双语开发者、国际化团队成员、多语言内容创作者
**场景**：在同一文档中混合使用英语、日语和中文拼音
**痛点**：
- 单一语言的拼写检查工具无法满足多语言需求
- 切换拼写检查语言繁琐且容易出错
**期望**：
- 拼写检查器能够自动识别并处理多种语言
- 同时支持英语技术术语、日语词汇和中文拼音的检查
- 提供统一的用户体验，无需手动切换语言设置

## 3. 高效写作与编码需求

**用户角色**：所有需要进行文本输入的VS Code用户
**场景**：在VS Code中编写代码、注释、文档或提交信息
**痛点**：
- 发现拼写错误后需要手动修改，影响写作流畅性
- 无法快速获取准确的拼写建议
**期望**：
- 实时检测拼写错误并提供视觉提示
- 点击拼写错误处即可获得准确的修复建议
- 支持一键修复功能，提高写作和编码效率

## 4. 自定义词汇管理需求

**用户角色**：专业领域工作者、团队协作成员
**场景**：在特定领域工作或团队项目中，需要使用领域特定术语或团队约定词汇
**痛点**：
- 无法方便地添加和管理自定义词汇
- 团队成员之间的自定义词汇无法共享
**期望**：
- 支持将特定术语添加到个人或团队词典
- 提供灵活的词典管理功能
- 支持与团队共享自定义词典
