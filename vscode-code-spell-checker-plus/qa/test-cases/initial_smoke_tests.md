初始冒烟测试用例

目的：快速验证扩展基本功能与多语言支持。

前置条件：
- 在本地机器上安装 VS Code
- 在仓库中准备好用于测试的示例文件（可以先使用 upstream 的 sampleWorkspaces）

用例 1：扩展能否在 VS Code 中以开发模式运行
步骤：
1. 打开扩展源代码（在 upstream 或本仓库）
2. 在 VS Code 中按 F5 启动扩展开发主机
3. 在开发主机中打开示例源码文件（如 README 或 sample 文件）
预期：
- 扩展激活且不报错
- 在示例文件中能检测到拼写问题（如果存在）

用例 2：中文注释（拼音）检测
步骤：
1. 在文件中添加中文注释，包含拼音式误拼写词或拼音缩写
2. 观察扩展是否能检测并给出建议
预期：
- 能检测到拼音拼写问题或正确忽略已配置词汇

用例 3：日文变量名检查
步骤：
1. 在示例源码中添加日文变量或注释
2. 观察扩展是否分词并检测拼写
预期：
- 能正确分词并检测出明显拼写问题

用例 4：一键修复交互
步骤：
1. 在出现拼写问题的代码处触发建议（右键或光标处）
2. 选择推荐候选词并应用
预期：
- 单词被替换，编辑器文件内容更新
- 能撤销替换操作
